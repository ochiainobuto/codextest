# 連番静止画からMP4動画を作成するWebアプリ

ブラウザ上で静止画の連番を読み込み、29.97fps (30000/1001fps) の MP4 動画に変換してダウンロードできるシングルページアプリです。すべての処理はブラウザ内で完結し、サーバーへファイルを送信することはありません。

## 利用方法

1. VS Code などの Ports ビューで `5500` 番ポートを公開し、外部ブラウザで `index.html` を開きます。
2. 連番になっている静止画をすべて選択してアップロードするか、画面上のエリアへドラッグ＆ドロップします。
3. 「MP4に変換してダウンロード」ボタンを押すと、ffmpeg.wasm を使用して 29.97fps の MP4 ファイルを生成します。
4. 変換が完了するとダウンロードリンクが表示され、生成された動画を保存できます。

## 備考

- すべてのフレームは同じ拡張子である必要があります (例: `.png` または `.jpg`)。
- 変換はブラウザのメモリを使用するため、大きなファイルや枚数が多い場合は時間がかかる、またはブラウザが応答しづらくなる可能性があります。
- ffmpeg.wasm の読み込みは初回のみ行われ、以降の変換では再読み込みされません。
- ブラウザが ES2015 構文や WebAssembly、Promise・Blob・URL.createObjectURL（または webkitURL の `createObjectURL` / `revokeObjectURL`）などの API に対応していない場合は、対応ブラウザへの切り替えを促すメッセージが表示されます。
- Codespaces などクロスオリジン分離が無効な環境では、ページが真っ白になる場合があります。その際は外部ブラウザで開き、Console に表示されるエラーを確認してください。
- favicon を空のデータ URI として同梱しているため、開発サーバーで `favicon.ico` の 404 エラーが出ることはありません。

## よくある質問

### Web 上で MP4 を作成できますか？

はい。本アプリは [ffmpeg.wasm](https://github.com/ffmpegwasm/ffmpeg.wasm) を利用し、選択した静止画をブラウザ内の仮想ファイルシステムへ書き込んでから `-framerate 30000/1001` で `libx264` を使って MP4 を生成し、`Blob` と `URL.createObjectURL` でダウンロードリンクを提供しています。ブラウザが WebAssembly や ES2015 の機能を備えていれば、追加のサーバー処理なしで MP4 を作成可能です。対応していないブラウザでは互換性メッセージが表示されるので、Chrome / Edge / Firefox / Safari など最新のブラウザをご利用ください。
